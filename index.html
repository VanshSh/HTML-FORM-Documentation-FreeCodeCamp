HTML Forms

An HTML form is used to collect user input. The user input is most often sent ot a server for processing.

Form is mostly built of following elements and attributes.

  • The <form> Element

    • It is used to create an HTML form for user input:

            
          <form>
          ..
          form elements
          ..
          </form>
  • The <input>Element

    • It is the most used form element.

            
          <input type="text"/"radio"/"checkbox"/"submit"/"button">

    We will discuss in deatil further in chapter.

  • The <label> Fields

    • This tag defines a label for many form elements. It is useful forscreen-reader users,because the screen-reader will read out loud the label when the user focus on the input element.

            
          <form>
          <label for="id"></label>
          <input type="text" id="for">
          </form>
    NOTE: "For" should be equal to "id".
  • The Name Attribute for <input>

    • Notice that each input field must have a "name" attribute to be submitted. If the "name" attribute is omitted, the value of input field will not be sent at all.

            
          <input type="text" name="name">

    Attributes

    Following are the mostly used attributes:

  • The Action Attribute

    • The "action" attribute defines the action to be performed when the form is submitted. Usually the form data is sent to a file on the server when the user clicks on the submit button.

            
          <form action="/action_page.php">
          .. 
          ..
          </form>
  • The Target Attribute

    • The "target attribute specifies where to display teh response that is received after submitting the form.

            
          <form action="/action.php" target="_blank"/"_self"/"_parent"/"_top"/"_self"></form>
  • The Method Attribute

    • The method attribute specifies the HTTP method to be used when submitting the form data.

            
          <form action="/action_page.php" method="get"/"post">
    NOTE: Never use "get" to send sensitive data!
  • The Autocomplete Attribute

    • The autocomplete attribute specifies whether a form should have autocomplete on or off.

            
          <from action="/action_page.php" autocomplete="on">
  • The Novalidate Attribute

    • The novalidate attribute is a boolean attribute. When present, it specifies that the form-data (input) should not be validated when submitted.

            
          <form action="/action_page.php" novalidate>

    Elements

    The HTML "form" element can contain one or more of the following form elements:

  • <select>

    • The "select" elements defines a drop-down list:

            
          <select id="cars" name="cars">
          <option value="volvo">Volvo</option>
          <option value="audi">Audi</option>
          <option value="Thar">Thar</option>
  • <textarea>

    • The textarea elements defines a multi-line input field:

            
          <textarea name="message" rows="10" cols="30">
          </textarea>
  • <legend> and <fieldset>

    • The "fieldset" element is used to group related data in a form.

    The "legend" element defines a caption for the "fieldset" element.

            
          <fieldset>
          <legend>Personalia</legend>
          ..
          </fieldset>

    Input Types

    This chapter describes the different types for the HTML "input" element:

  • <checkbox>

    • It let a user select ZERO or MORE options of a limited number of choices.

            
          <input type="checkbox" id="vehicle1" name="vehicle1">
          <input type="checkbox" id="vehicle2">
          <input type="chrckbox" id="vehicle3">
  • <password>

    • It defines a "password" field.

            
          <input type="password" id="pwd" name="pwd">
  • <submit>

    • It defines a button for submitting form data to a form-handler.

            
          <input type="submit" value="submit">

    Input Attributes

    This chapter describes the different attributes for the HTML "input" element:

  • <readonly>

    • The input "readonly" attributes specifies that an input field is read-only.

            
          <input type="text" name="fname" value="john" readonly>
  • <disabled>

    • The input "disabled" attribute specifies that an input field should be disbaled.

            
          <input type="text" id="fname" name="fname" value="john" disabled>
  • <size>

    • The input size attributes specifies the visible width, in characters, of an input field.

            
        <input type="text" id="pin" name="pin" size="4">
